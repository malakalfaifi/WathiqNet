How It Works
  1. Select a network interface to capture from
  2. Optionally set a BPF filter to capture specific traffic
  3. Click Start Capture to begin monitoring
  4. Packets are analyzed in real-time by the detection engine
  5. Alerts are generated for suspicious patterns
  6. All data is saved to the database for later analysis
Important Notes
  • Administrator privileges required - Run the application as admin
  • High traffic networks - Use filters to reduce data volume
  • Database storage - Captured packets are stored, monitor disk usage
  • Performance - Capturing may impact network performance on busy networks
Example BPF Filters:
  • tcp port 80 - HTTP traffic only
  • host 192.168.1.100 - Traffic to/from specific host
  • not port 22 - Exclude SSH traffic
  • tcp and port 443 - HTTPS traffic
